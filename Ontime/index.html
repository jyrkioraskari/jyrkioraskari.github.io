Abstract

The Ontology for Open Mobility Data provides a structured framework to link external mobility datasets with Just-in-Time (JIT) construction logistics concepts. Its goal is to achieve semantic alignment between diverse open mobility sources and lean construction needs. Rather than modeling internal construction processes, it maps external data—such as traffic, transport, and mobility information—to relevant JIT concepts. By defining shared semantics, the ontology enables interoperability and data reuse. It is built on a manually curated baseline and refined using large language model (LLM) embeddings to cluster domain terms, define scope, identify datasets, and suggest reusable components.

Introduction back to ToC

The Ontology for Open Mobility Data offers a structured semantic framework for representing external mobility datasets and connecting them to Just-in-Time (JIT) construction logistics concepts. Its main goal is to facilitate semantic alignment between diverse open mobility data sources and the internal logistics needs of lean construction workflows. Instead of modeling the internal construction domain itself, the ontology concentrates on mapping external data sources—such as traffic, transport, and mobility information—to relevant JIT logistics concepts. By establishing shared semantics, it promotes data interoperability and reuse, enabling construction stakeholders to incorporate real-time mobility information into operational decision-making. This approach is especially useful in scenarios where route restrictions, traffic conditions, and external events impact construction logistics. The ontology is built from a manually curated baseline and enhanced through large language model (LLM) embeddings. These embeddings help cluster domain terms, analyze scope, identify relevant datasets, and suggest reusable ontology components. This hybrid method combines expert-driven structure with AI-supported scalability.
